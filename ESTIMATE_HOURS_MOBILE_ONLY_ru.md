# Оценка в часах — Mobile App (отдельно)
## Pilates Space (iOS + Android, React Native)

**Считаем отдельно от Web/Admin.**  
**Scope mobile:** клиентское приложение (без админки), интеграция с уже готовым backend API.

---

## Что входит в mobile scope
- Авторизация (signup/login/refresh/logout)
- Профиль клиента
- Расписание занятий
- Бронирование/отмена
- Waitlist (join/confirm/leave + таймер подтверждения)
- Покупка продуктов и оплата через backend (JCC flow)
- Экран баланса пакетов и истории посещений
- Push-уведомления (booking/waitlist/payment)
- QA, стабилизация, публикация в TestFlight/Internal testing

---

## Оценка по блокам

### 1) Foundation
- Проект RN, навигация, state/data layer, env-конфиги: **24–36ч**
- Дизайн-система/базовые UI-компоненты: **16–26ч**

**Итого:** **40–62ч**

### 2) Auth + Profile
- Экран логина/регистрации/восстановления: **20–32ч**
- Профиль пользователя + редактирование: **10–18ч**

**Итого:** **30–50ч**

### 3) Schedule + Booking
- Календарь/список занятий + фильтры: **22–36ч**
- Бронирование/отмена + UX статусы: **16–28ч**

**Итого:** **38–64ч**

### 4) Waitlist
- Join/leave/confirm flow: **16–28ч**
- Таймер подтверждения + edge cases: **10–18ч**

**Итого:** **26–46ч**

### 5) Purchases + Payment UX
- Каталог продуктов (разовое/пакеты/инд): **14–24ч**
- Checkout flow + статусы платежа: **20–34ч**

**Итого:** **34–58ч**

### 6) Balance + History
- Баланс пакета, история посещений/покупок: **14–24ч**

**Итого:** **14–24ч**

### 7) Push Notifications
- Интеграция APNs/FCM token flow: **12–20ч**
- Обработка событий и deep links: **10–18ч**

**Итого:** **22–38ч**

### 8) QA + Release prep
- E2E/manual regression критических сценариев: **24–40ч**
- Fixes/hardening/perf: **20–36ч**
- Store prep (иконки, privacy, сборки): **12–20ч**

**Итого:** **56–96ч**

---

## Сводная оценка
- **Оптимистично:** **260–300ч**
- **Реалистично:** **320–380ч**
- **С буфером:** **400–460ч**

---

## Сроки (1 mobile разработчик)
- **Реалистично:** 8–11 недель
- **С буфером и review cycles:** 11–13 недель

---

## Важные зависимости
- Готовность backend API и стабильных контрактов (OpenAPI)
- Готовность push-инфры на backend
- Доступы в Apple Developer / Google Play Console
- Тестовые платежные сценарии через backend (JCC)
