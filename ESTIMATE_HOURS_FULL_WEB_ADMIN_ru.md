# Оценка в часах (FULL) — Web + Admin, 1 разработчик
## Pilates Space (16–20 недель)

**Предпосылка:** 1 full-time разработчик, 35–40 продуктивных часов/неделя.  
**Итоговый коридор:** **560–800 часов**.

---

## 1) Discovery + архитектура (Week 1–2)
- Уточнение требований, use-cases, границы MVP: 20–28ч
- ERD + доменная модель: 12–18ч
- OpenAPI черновик + контракты: 14–22ч
- Поднятие окружений, CI/CD baseline: 16–24ч

**Итого:** **62–92ч**

---

## 2) Backend Core (Week 3–5)
- Auth + RBAC: 20–30ч
- Клиенты/тренеры/локации CRUD: 22–32ч
- Расписание/сессии/емкость: 28–40ч
- Бронирования/отмены/cutoff: 28–42ч
- Продукты/пакеты/ledger списаний: 28–42ч
- Аудит-лог каркас: 10–16ч

**Итого:** **136–202ч**

---

## 3) Web (публичный + кабинет) (Week 6–7)
- Публичные страницы: 18–30ч
- Auth UI (signup/login/reset): 14–22ч
- Расписание + бронь/отмена: 24–36ч
- Профиль/баланс/история: 16–26ч
- Локализация EN/RU: 8–14ч

**Итого:** **80–128ч**

---

## 4) Кастомная админка (Week 8–10)
- Dashboard и оперативные виджеты: 16–26ч
- Клиентская карточка и история: 20–32ч
- Управление расписанием и группами: 24–36ч
- Продажи (разовое/пакеты/инд): 20–32ч
- Кассовые операции (cash статусы): 14–22ч
- Аудит-экран: 10–18ч

**Итого:** **104–166ч**

---

## 5) JCC + POS initiation (Week 11–12)
- JCC API client + маппинг статусов: 18–30ч
- Инициация суммы на терминал из админки: 16–26ч
- Callback/webhook обработка: 14–22ч
- Polling fallback + retry/idempotency: 12–20ч
- Автоактивация покупки + payment events: 14–24ч

**Итого:** **74–122ч**

---

## 6) Waitlist engine (Week 13)
- FIFO очередь + join/leave/confirm: 14–24ч
- Таймеры и авто-передача слота: 12–20ч
- Админ-override + аудит: 8–14ч

**Итого:** **34–58ч**

---

## 7) Push notifications (Week 14)
- Notification service + шаблоны: 10–18ч
- События booking/waitlist/payment: 10–18ч
- Retry/dedup/log delivery: 10–16ч

**Итого:** **30–52ч**

---

## 8) QA/UAT/Hardening (Week 15–16)
- Integration/E2E тесты критических флоу: 24–40ч
- Багфикс и стабилизация: 24–40ч
- Мониторинг/алерты/Sentry: 10–18ч
- Backup/restore check + release prep: 8–16ч

**Итого:** **66–114ч**

---

## 9) Буфер Week 17–20 (по факту)
- UAT-правки/регрессии/UX polishing: **0–90ч**

---

## Сводка
- **Оптимистично (без сильных блокеров):** ~**560–620ч**
- **Базовый реалистичный:** ~**650–720ч**
- **С буфером и доработками после UAT:** ~**740–800ч**

---

## Для сметы
Если нужна коммерческая вилка, формула:

`Бюджет = Часы × Ставка`

Пример при ставке 50€/ч:
- 650ч → 32,500€
- 720ч → 36,000€
- 800ч → 40,000€
